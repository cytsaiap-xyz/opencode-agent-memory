# REFLECT Stage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `distill reflect` — cross-session consolidation per spec `docs/superpowers/specs/2026-07-11-reflect-design.md`: deterministic clustering → LLM insight/merge/none ops (judge-gated) → promotion queue, all through existing governance, stateless-idempotent.

**Architecture:** `distiller/cluster.ts` (pure clustering) + `distiller/reflect.ts` (ops, prompts, promotion scan, idempotency skips) + CLI command + scheduling hooks. Merge ops ride the EXISTING supersession path (policy interception inherited free); promotions ride the EXISTING review queue.

## Global Constraints

- Clustering deterministic and pure: domain grouping → pairwise `titleJaccard`-style similarity over `title + " " + trigger` (reuse pool.ts tokenization) ≥ 0.35 → connected components, size ≥ 2, cap 12 members (keep highest-confidence members when capping), clusters processed largest-first.
- Insight entries must clear the judge gate (reuse `judgeCandidate`, judges option, median ≥ salienceMin) — an insight is held to the same bar as any candidate.
- ALL mutations go through existing machinery: merges call the same supersession code (decision/convention targets → review queue automatically); promotions create pending copies in quarantine (`review: human_pending`).
- New optional frontmatter field `promoted_from: string | null` (same optional-parse pattern as `supersedes`; serializer always writes it; missing → null; spec amendment noted in the reflect spec by Task 1). `approveEntry` on an entry with `promoted_from` appends note `promoted to <approved id>` to the source entry (drift-tolerant: missing source → warning).
- Statelessness: re-running reflect is a no-op — skips per spec (insight exists citing the same member set: detected via `derived from:` note listing member ids; merge members already superseded; promotion copy pending/active already citing source via `promoted_from`).
- `--dry-run` writes NOTHING (assert via mtime/file-count sweep in tests) and prints planned ops.
- LLM calls: cluster ops + judge gating only; FakeLlm in tests.
- Commits: conventional + standard trailers:
  `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>` and
  `Claude-Session: https://claude.ai/code/session_01ALAk5sCENNGXZyy6mSg8tF`

### Task 2: distiller/cluster.ts — deterministic clustering (pure)

**Files:** Create `distiller/cluster.ts`; Test `distiller/cluster.test.ts`

**Interfaces:**
```ts
import type { MemoryEntry } from "./types"
export function pairSimilarity(a: MemoryEntry, b: MemoryEntry): number  // token-set Jaccard over `${title} ${trigger}`, pool.ts tokenization + degenerate-title rule
export interface Cluster { domain: string; members: Array<{ entry: MemoryEntry; path: string }> }
export function buildClusters(entries: Array<{ entry: MemoryEntry; path: string }>, opts?: { threshold?: number; minSize?: number; cap?: number }): Cluster[]
```
- Group ACTIVE entries by each domain tag (an entry with 2 domains can appear in 2 groups; a cluster is deduped so the same member set under two domains yields ONE cluster — key = sorted member ids).
- Within a group: union-find over pairs with similarity ≥ threshold (0.35); components with ≥ minSize (2) members become clusters; cap (12) keeps highest-confidence members; output sorted largest-first, deterministic tie-break by sorted-member-id string.

Tests: similarity boundary 0.35, component transitivity (A~B~C chains cluster even when A≁C), multi-domain dedup by member set, cap keeps top-confidence, ordering determinism, non-active excluded by caller contract (buildClusters trusts input; test the CALLER filter in Task 3).
Commit: `feat(distiller): deterministic memory clustering for reflect`

Flow: collect active entries (via index.search? NO — walk `listEntryPaths` + parse, filter active + optional project; index-agnostic) → buildClusters → per cluster: idempotency skip (an ACTIVE entry exists whose notes contain `derived from: <sorted member ids joined>` exact string) → LLM op → parse (throw → errors++, continue) →
- insight: build Candidate-shaped object → judge gate (judges>1) → below bar → skipped++ → else create entry (id via entryId, `memory_class: "semantic"`, type from op, project = members' common project else "global", evidence union deduped by session, confidence via computeConfidence over distinct sessions, `review: "auto"`, status active, notes: [`<date>: derived from: <sorted member ids>`], `promoted_from: null`, `supersedes: null`, judgeNote appended when judged) → writeEntry + upsert (dry-run: log only).
